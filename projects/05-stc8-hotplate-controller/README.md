# 基于 STC8G1K08A 的双单片机加热台控制系统

## 项目概述

| 属性 | 内容 |
|------|------|
| **开发时间** | 2026.04 |
| **项目类型** | 双单片机加热台控制板开发与联调 |
| **主控方案** | `STC8G1K08A` × 2 |
| **开发环境** | `Keil C51` |
| **下载工具** | `STC-ISP` |
| **项目角色** | 独立开发与联调 |

本项目是一套面向加热台控制板的双单片机应用系统。  
系统将人机交互与温控执行拆分到两颗 `STC8G1K08A` 上，分别负责界面操作和实际控温，从而降低单芯片资源压力，并让界面逻辑与控温逻辑能够独立调试。

## 技术栈

- **MCU**：`STC8G1K08A`
- **开发环境**：`Keil C51`
- **下载方式**：`STC-ISP` 串口下载
- **系统时钟**：内部 `IRC 24MHz`
- **串口参数**：`115200 8N1`
- **显示模块**：`0.91 寸 OLED`，软件模拟 `IIC`
- **温度采样**：`100K 3950 NTC`
- **加热驱动**：`MOC3041SR2M + BT138-600E`
- **控温算法**：整数 `PID` + 时间比例控制

## 系统架构

```text
按键 / OLED / 风扇
        │
        ▼

        ├── 温度标定修正

```

## 软件设计

### 核心功能
1. **双单片机分层控制**  
   芯片B负责参数设定与显示，芯片A负责实时测温与控温执行，避免界面刷新和控温主循环互相干扰。

2. **串口协议联动**  
   芯片B向芯片A发送目标温度和运行状态，芯片A按固定周期返回当前温度、风扇状态、加热状态和运行状态。

3. **NTC 自动量程测温**  
   芯片A通过切换不同上拉网络覆盖较宽温区，兼顾低温和高温段的分辨率。

4. **温度标定修正**  
   在基础查表换算之上，根据实测数据对显示温度和控温温度做统一修正，减少探头个体差异带来的偏差。

5. **PID + 时间比例控温**  
   PID 输出为功率百分比，再映射成 `1s` 时间窗口内的导通段数，适配交流加热场景。

6. **风扇联动控制**  
   停热后根据温度回差自动开启风扇散热，降低余热影响。

### 调试内容
- 双芯片串口互联调试
- OLED 显示联调
- ADC 按键识别调试
- NTC 采样与量程切换验证
- 温度标定修正
- PID 参数整定
- 加热输出链路验证
- 风扇联动验证

### 关键技术
1. **双 MCU 协议拆分**  
   将界面更新与控温执行拆分到两颗 8051 内核单片机上，减轻单芯片实时压力。

2. **整数控温实现**  
   在 `8KB` 程序空间限制下，采用整数查表、整数标定和整数 PID 方式完成温度闭环控制。

3. **交流加热时间比例控制**  
   放弃高频 PWM，改为低频时间比例输出，更适合可控硅交流加热场景。

4. **现场标定闭环**  
   通过程序显示温度与外部测温仪对照，不断修正测温模型，使显示与控温基准逐步逼近真实温度。

## 通信协议

### 芯片B -> 芯片A
```text
C,<set_temp>,<run>\r\n
```

示例：
```text
C,230,1
```

### 芯片A -> 芯片B
```text
S,<set_temp>,<cur_temp>,<fan>,<heat>,<run>\r\n
```

示例：
```text
S,230,185,1,0,1
```

### 应答帧
```text
OK\r\n
```
