# 基于 MSPM0G3507 的竞赛控制小车系统演化开发

## 项目概述

| 属性 | 内容 |
|------|------|
| **开发时间** | 2025.04 - 2025.07 |
| **主控芯片** | TI MSPM0G3507 |
| **开发环境** | Code Composer Studio |
| **项目角色** | 独立开发 |
| **项目定位** | 竞赛控制小车平台迭代开发 |

## 项目简介

这个项目不是一次成型的单一功能小车，而是一套围绕竞赛任务不断迭代的控制平台。工程从基础循迹底盘开始，逐步加入舵机、步进云台、电磁铁、总线云台、IMU 和多路串口通信，后期进一步接入树莓派 / OpenCV 侧视觉结果，用于配合车体完成更复杂的任务流程。

从现有版本演进和源码内容来看，这个项目更适合作为一套竞赛控制小车平台来展示。它体现的重点不只是“某一个题目的最终形态”，而是从底盘控制到执行机构，再到视觉协同的一步步完善过程，最终已经接近竞赛使用小车的完整形态。

## 技术栈

- **MCU**：TI MSPM0G3507
- **开发环境**：Code Composer Studio、SysConfig
- **底盘驱动**：TMC2209 步进电机驱动
- **传感器**：灰度循迹、IMU
- **执行机构**：PWM 舵机、总线云台、电磁铁
- **通信接口**：多路 UART
- **视觉协同**：树莓派 / OpenCV 串口结果接入
- **显示与调试**：LCD、串口调试输出

## 演化过程

### v1.0 基础循迹完成

这一阶段先把小车底盘跑起来，重点完成灰度循迹和基础运动控制。

- 完成灰度传感器循迹
- 建立步进底盘控制链路
- 初步完成 PID 参数调试
- 形成最早可运行的小车底盘版本

### v1.1 添加舵机、云台、电磁铁

在能稳定跑线的基础上，开始加入执行机构，让小车从单纯运动平台变成可以做任务动作的平台。

- 加入 PWM 舵机控制
- 加入步进云台相关控制
- 加入电磁铁执行机构
- 开始处理循迹与动作配合问题

### v1.2A 总线云台与 IMU 接入

这一阶段开始明显向更完整的竞赛控制平台演进，除了底盘本身，还加入姿态和更稳定的执行机构控制。

- 云台从早期方案继续升级
- 加入总线舵机云台控制
- 接入 IMU 串口姿态数据
- 增强原地转向和姿态调整能力

### v1.2B 接入树莓派视觉通信

后期版本已经不再只是底盘和执行机构控制，而是进入“视觉协同”阶段。源码里可以看到主控通过串口接收 `OpenCv_Rx` 数据，并在流程中配合显示和任务判断使用。

- 接入树莓派 / OpenCV 串口数据
- 保留形状、颜色结果接收与显示逻辑
- 保留底盘循迹、云台动作、串口协同的综合控制结构
- 形成接近竞赛任务使用的小车控制平台

### 底盘控制

- 灰度循迹
- 圈数 / 黑块 / 转向状态判断
- 步进电机左右轮速度控制
- 基于 PID 的循迹调节

### 执行机构

- PWM 舵机控制
- 总线舵机双舵机控制
- 电磁铁开关控制
- 云台左右动作预置

### 姿态与通信

- IMU 串口姿态数据解析
- 原地转向 PID 调整
- 多路 UART 分工使用
- 串口调试输出

### 视觉协同

- 接收树莓派 / OpenCV 侧串口结果
- 保存 `OpenCv_Rx` 数据
- 在 LCD 上显示形状 / 颜色结果
- 在任务流程中等待并使用视觉侧返回信息

## 项目特点

- 不是单一版本工程，而是完整保留了多个迭代阶段
- 可以直接看到从“基础循迹”到“视觉协同控制”的演化路径
- 同时覆盖底盘控制、执行机构控制、姿态处理和串口协同
- 能体现 TI MSPM0 平台在竞赛控制类项目中的实际开发过程
