# 编译说明

## 开发环境要求

### 软件要求
- **操作系统**：Windows 10/11 或 Linux
- **IDE**：TI Code Composer Studio (CCS) v12.0 或更高版本
- **配置工具**：SysConfig（集成在CCS中）
- **驱动库**：TI DriverLib（已包含在项目中）
- **编译器**：TI ARM Clang Compiler（随CCS安装）

### 硬件要求
- **开发板**：TI MSPM0G3507开发板
- **调试器**：XDS110调试器（推荐）
- **计算机**：至少4GB RAM，10GB可用存储空间

## 工程导入

### 方法一：直接导入CCS工程
1. 打开Code Composer Studio
2. 选择 `File` → `Import`
3. 选择 `Code Composer Studio` → `Existing CCS/CCE Eclipse Projects`
4. 点击 `Browse` 选择项目根目录
5. 确保 `Stepping_Trace_1.2` 项目被选中
6. 点击 `Finish` 完成导入

### 方法二：新建工程后复制文件
1. 在CCS中新建MSPM0G3507工程
2. 将项目文件复制到工程目录
3. 刷新工程资源

## 硬件配置

### SysConfig配置
1. 双击工程中的 `Stepping_Trace.syscfg` 文件
2. SysConfig工具将自动打开
3. 确认以下配置：

#### 引脚配置
| 外设 | 引脚 | 功能 |
|------|------|------|
| UART0 | PA0(TX), PA1(RX) | 调试输出 |
| UART1 | PA2(TX), PA3(RX) | 树莓派通信 |
| I2C0 | PA6(SCL), PA7(SDA) | 传感器通信 |
| SPI0 | PB2(MOSI), PB3(MISO), PB4(SCLK), PB5(CS) | LCD显示 |
| PWM0 | PB10 | 步进电机控制 |
| GPIO | PB17 | 电磁铁控制 |

#### 时钟配置
- **系统时钟**：64MHz
- **外设时钟**：与系统时钟同步
- **调试时钟**：使能

#### 外设实例
- UART实例：4个（UART0-3）
- 定时器实例：2个（Trace_Timer_INST, TIMER_0_INST）
- PWM实例：5个（舵机和电机控制）
- GPIO实例：7个（各种IO控制）
- I2C实例：1个（传感器通信）

### 配置生成
1. 完成配置后点击 `Save`
2. SysConfig将自动生成驱动代码
3. 生成的代码位于 `syscfg/` 目录

## 编译步骤

### 首次编译
1. 确保工程已正确导入
2. 右键点击工程名称
3. 选择 `Build Project` 或按 `Ctrl+B`
4. 等待编译完成

### 常见编译问题

#### 问题1：找不到头文件
- **症状**：`fatal error: 'driverlib.h' file not found`
- **原因**：DriverLib路径未正确配置
- **解决**：
  1. 右键工程 → `Properties`
  2. 选择 `Build` → `ARM Compiler` → `Include Options`
  3. 添加DriverLib包含路径

#### 问题2：链接错误
- **症状**：`undefined reference` 错误
- **原因**：库文件未链接
- **解决**：
  1. 右键工程 → `Properties`
  2. 选择 `Build` → `ARM Linker` → `File Search Path`
  3. 添加DriverLib库文件路径

#### 问题3：内存不足
- **症状**：`region ROM overflowed` 错误
- **原因**：代码体积超过Flash容量
- **解决**：
  1. 优化代码，移除调试信息
  2. 启用编译器优化选项
  3. 检查是否有不必要的库链接

### 优化编译选项

#### 编译器优化
1. 右键工程 → `Properties`
2. 选择 `Build` → `ARM Compiler` → `Optimization`
3. 优化级别建议：
   - 调试阶段：`-O0`（无优化）
   - 发布阶段：`-O2`（平衡优化）

#### 链接器配置
1. 右键工程 → `Properties`
2. 选择 `Build` → `ARM Linker` → `Basic Options`
3. 堆栈大小设置：
   - 堆大小：`0x400`（1KB）
   - 栈大小：`0x800`（2KB）

## 下载与调试

### 程序下载
1. 连接XDS110调试器到开发板
2. 确保开发板供电正常
3. 右键工程 → `Debug As` → `Code Composer Debug Session`
4. 程序将自动下载并进入调试模式

### 调试配置

#### 调试器设置
1. 右键工程 → `Debug Configurations`
2. 选择 `Code Composer Debug Session`
3. 在 `Main` 标签页确认：
   - Project: `Stepping_Trace_1.2`
   - Configuration: `Debug`
4. 在 `Target` 标签页确认调试器连接

#### 调试视图
- **反汇编视图**：查看机器指令
- **寄存器视图**：查看CPU寄存器状态
- **内存视图**：查看内存内容
- **变量视图**：查看和修改变量值
- **表达式视图**：计算表达式值

### 实时调试功能

#### 断点设置
1. 在代码行号左侧双击设置断点
2. 程序执行到断点处将暂停
3. 可查看此时的所有状态

#### 单步调试
- **Step Into (F5)**：进入函数内部
- **Step Over (F6)**：执行当前行，不进入函数
- **Step Return (F7)**：执行到函数返回
- **Resume (F8)**：继续执行到下一个断点

#### 观察点
1. 右键变量 → `Breakpoints` → `Hardware Watchpoint`
2. 当变量值改变时程序暂停
3. 适用于调试数据变化问题

2. 选择 `Debug` 或 `Release`
3. 重新编译工程

## 常见问题解答

### Q1：编译速度慢怎么办？
- **A**：
  1. 关闭不必要的IDE插件
  2. 增加编译器工作内存
  3. 使用增量编译而非全量编译

### Q2：如何减少代码体积？
- **A**：
  1. 移除未使用的函数和变量
  2. 使用编译器链接时优化
  3. 选择适当的优化级别

### Q3：调试时变量值显示不正确？
- **A**：
  1. 确保使用Debug配置编译
  2. 检查优化级别是否为-O0
  3. 变量可能被编译器优化掉，标记为volatile

### Q4：程序下载失败？
- **A**：
  1. 检查调试器连接是否正常
  2. 确认开发板供电正常
  3. 检查芯片型号选择是否正确
  4. 尝试复位开发板重新连接

## 性能优化建议

### 编译时优化
1. **函数内联**：对小型函数使用 `inline` 关键字
2. **常量优化**：将常量定义为 `const`
3. **循环优化**：避免循环内部复杂计算
4. **内存对齐**：重要数据结构按4字节对齐

### 运行时优化
1. **中断优化**：减少中断服务程序执行时间
2. **内存优化**：使用内存池代替动态分配
3. **算法优化**：选择时间复杂度低的算法
4. **数据优化**：使用适当的数据类型
